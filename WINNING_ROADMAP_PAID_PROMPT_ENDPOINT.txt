THREE SMART CONTRACTS (all built with AlgoKit Python SDK):

  CONTRACT 1 — PromptRegistry
    - Stores: prompt_id, creator_wallet, base_price, usage_count,
              creator_stake, prompt_hash (NOT the prompt itself)
    - Bonding curve: price = base_price * (1 + usage_count / 500)
    - Creator must stake minimum 2 ALGO to publish
    - Stake slashed if quality_score drops below threshold

  CONTRACT 2 — PaymentGate
    - Receives ALGO payment from user
    - Verifies amount >= current bonding curve price
    - Emits a one-time SessionToken (ARC-4 struct, expires in 60s)
    - Splits payment: 85% to creator, 10% to protocol treasury,
      5% burned (deflationary, judges love this detail)
    - Logs transaction hash for audit trail

  CONTRACT 3 — ChainOrchestrator
    - Accepts a list of prompt_ids as input (the "chain")
    - Calls PaymentGate for each prompt in sequence
    - Holds funds in escrow until all hops confirm
    - If any hop fails, refunds entire chain atomically
    - This is the DeFi composability angle — atomic multi-hop

SMART CONTRACT SKELETON (prompt_registry.py):

  from algopy import (
      ARC4Contract, arc4, Global, Txn, UInt64, String
  )

  class PromptRegistry(ARC4Contract):
      base_price: arc4.UInt64        # microALGO
      usage_count: arc4.UInt64
      creator: arc4.Address
      creator_stake: arc4.UInt64
      prompt_hash: arc4.StaticArray[arc4.Byte, typing.Literal[32]]

      @arc4.abimethod(create="require")
      def initialize(
          self,
          base_price: arc4.UInt64,
          prompt_hash: arc4.StaticArray[arc4.Byte, typing.Literal[32]]
      ) -> None:
          self.base_price = base_price
          self.usage_count = arc4.UInt64(0)
          self.creator = arc4.Address(Txn.sender)
          self.prompt_hash = prompt_hash

      @arc4.abimethod
      def current_price(self) -> arc4.UInt64:
          # Bonding curve: price rises with usage
          multiplier = 1000 + (self.usage_count.native // 500)
          return arc4.UInt64(
              (self.base_price.native * multiplier) // 1000
          )

      @arc4.abimethod
      def increment_usage(self) -> None:
          # Only callable by PaymentGate contract
          assert Txn.sender == PAYMENT_GATE_ADDRESS
          self.usage_count = arc4.UInt64(
              self.usage_count.native + 1
          )

PAYMENT GATE SKELETON (payment_gate.py):

  class PaymentGate(ARC4Contract):

      @arc4.abimethod
      def pay_and_issue_token(
          self,
          payment: gtxn.PaymentTransaction,
          prompt_id: arc4.UInt64,
          registry_app: arc4.Application
      ) -> arc4.DynamicBytes:
          # 1. Get current bonding curve price from registry
          current_price = arc4.arc4_signature("current_price()(uint64)")
          # 2. Verify payment >= current price
          assert payment.amount >= current_price
          assert payment.receiver == Global.current_application_address
          # 3. Split payment (85/10/5)
          creator_share = (payment.amount * 85) // 100
          protocol_share = (payment.amount * 10) // 100
          # 4. Issue 32-byte session token (expires in 60 blocks)
          session_token = sha256(
              Txn.tx_id + Global.latest_timestamp.bytes
          )
          # 5. Increment usage on registry
          # 6. Return session token to caller
          return arc4.DynamicBytes(session_token)

WHAT VIBEKIT DOES IN YOUR SYSTEM:

  VibeKit is your autonomous agent framework. It is the "brain"
  that orchestrates the entire flow without human intervention
  after the user clicks "Run Chain."

THREE AGENT MODULES:

  MODULE 1 — ModelRouter
    Purpose: Decides which AI model to use for each prompt in
    the chain based on task type, input length, and live cost.

    Decision logic:
      if task_type == "reasoning" or "analysis":
          use Claude 3.5 Sonnet (best reasoning)
      elif task_type == "creative" or "writing":
          use GPT-4o (best creative output)
      elif input_length > 50000 chars:
          use Gemini 1.5 Pro (best context window)
      elif cost_priority == "cheap":
          use Claude Haiku or GPT-4o-mini

    Why judges love this: The agent is making real decisions,
    not just calling one fixed API. This IS agentic commerce.

  MODULE 2 — ChainExecutor
    Purpose: Executes a sequence of prompts where output of
    prompt N becomes the input to prompt N+1. Handles all
    payments autonomously across the chain.

    Execution flow:
      1. Receive chain: [prompt_A, prompt_B, prompt_C]
      2. Calculate total ALGO cost upfront (with bonding curves)
      3. Submit to ChainOrchestrator contract for atomic escrow
      4. Execute prompt_A → get response_A
      5. Feed response_A as context into prompt_B → get response_B
      6. Feed response_B as context into prompt_C → get response_C
      7. Release escrow, stream ALGO to all creators
      8. Return combined response to user

    DEMO CHAIN (use this exact chain in your demo):
      Prompt A: "Startup Pitch Analyzer" — analyze user's idea
      Prompt B: "Investor Objection Handler" — anticipate VCs
      Prompt C: "Cold Email Generator" — write outreach email
      Result: User gets a full investor package in one click.
      Total cost: ~1.2 ALGO, 3 creators earn simultaneously.

  MODULE 3 — WalletManager
    Purpose: Manages signing, submission, and confirmation of
    Algorand transactions. Handles retry logic if confirmation
    takes >10 seconds. Uses AlgoKit's AtomicTransactionComposer
    for grouping multiple txns into one atomic batch.

    Key implementation:
      from algokit_utils import AtomicTransactionComposer, ABIMethod
      atc = AtomicTransactionComposer()
      atc.add_method_call(...)  # add all chain payments at once
      result = atc.execute(algod_client, wait_rounds=4)
      # All txns confirmed or none go through — atomic guarantee

     SHA-256 hash is. The actual prompt is AES-encrypted in our
     backend and only decrypted after the session token from the
     contract is verified. So even we can't access prompts
     without a valid on-chain payment proof."

  4. "VibeKit's model router uses real-time token pricing from
     each AI provider's API to select the cheapest model that
     meets the task requirements. Users always pay a fixed ALGO
     price but get dynamically optimized AI behind the scenes."

  5. "5% of every payment is sent to an unspendable burn address
     on Algorand. This creates genuine deflation as the protocol
     grows — a built-in incentive for early prompt creators."

  6. "Creator staking creates skin-in-the-game. If a creator
     publishes a low-quality prompt and users rate it poorly,
     their staked ALGO is slashed. Quality is economically
     enforced, not socially enforced."
